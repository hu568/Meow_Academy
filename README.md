# 喵学堂 (Meow Academy) 🐱

> **AI 驱动的个人学习系统**

一个基于多 Agent 协作的智能学习平台，整合知识管理、个性化教学、智能出题和学习档案功能。

---

## 📚 项目简介

喵学堂不是传统软件项目，而是一个**知识库 + 多 Agent 协作系统**。所有内容均以 Markdown 形式组织，由 AI 智能体协调运作，实现个性化的学习体验。

### 核心特点

- 🤖 **多 Agent 智能协作**：4 个专业化 AI 角色分工明确
- 📖 **结构化知识管理**：学科模板化，支持无限扩展
- 🧠 **双存储架构**：SQLite 数据库 + Markdown 视图
- 🎯 **个性化学习路径**：基于学习档案的智能推荐

---

## 🏗️ 系统架构

```
喵学堂/
├── 📥 1. 知识层/              ← 核心知识资产
│   └── 结构化知识库/
│       ├── _学科模板_/         ← 格式化书页、知识点卡片、专业词典
│       └── {学科}/             ← 数学、物理、模拟电路等实例
│
├── 🧠 2. 智能体层/            ← 多 Agent 协作系统
│   └── .opencode/agents/      ← Agent 角色定义
│
├── 👤 3. 用户层/              ← 学习者画像与档案
│   └── （规划中：会话状态、错题本、学习轨迹）
│
└── 🗄️ 4. 数据层/              ← SQLite + Markdown 双存储
    ├── 喵学堂.db               ← 数据源（AI 操作）
    └── 索引/图书总录.md         ← 只读视图（人类阅读）
```

## 🎭 Agent 角色体系

| Agent | 角色 | 职责 | 模式 |
|-------|------|------|------|
| **西西喵斯** (Sisyphus) | 中央调度 | 意图识别、任务分派、工作流编排 | `primary` |
| **帕秋莉** (Patchouli) | 图书管理员 | 资料解析、知识归档、OCR 结构化 | `subagent` |
| **月咏小萌** (Komoe) | 授课老师 | 知识讲解、答疑解惑、教学引导 | `subagent` |
| **茅场晶彦** (Kayaba) | 出题考官 | 练习生成、测试设计、错题分析 | `subagent` |

### 协作流程

```
用户提问 → 西西喵斯识别意图 → 分派子 Agent → 执行 → 整合回复
                              ↓
                    帕秋莉 / 小萌 / 晶彦
```

## 📂 知识库结构

### 学科模板 (`知识层/结构化知识库/_学科模板_/`)

每个学科包含三类内容：

```
{学科}/
├── 📄 格式化书页/
│   └── {书名}_{页码}.md          ← 保留完整章节结构的整书 Markdown
├── 🧩 知识点卡片/
│   └── {知识点ID}_{名称}.md       ← 原子化，可复用（含 Frontmatter）
└── 📖 专业词典/
    └── {词条}.md                  ← 专业术语定义
```

### 当前学科

- **数学** - 向量数量积等
- **模拟电路** - 二极管、PN结、稳压管等

| 层级 | 技术 | 用途 |
|------|------|------|

## 🚀 使用方式

### 入口点

与 **西西喵斯**（主智能体）对话即可开始：

- 📤 **导入资料** → "帮我整理这本教材"
- 🎓 **学习知识** → "讲解二极管的伏安特性"
- ✏️ **练习测试** → "给我出几道向量相关的题目"
- 📊 **查看档案** → "我最近的学习进度如何"

### 数据流向

```
AI/程序操作 → 写入 SQLite → 自动生成 Markdown
                    ↑
人类阅读 ←──── 查看索引/图书总录.md（只读）
```

---

## ⚠️ 注意事项

1. **不要手动编辑 `索引/图书总录.md`** - 此文件由数据库自动生成
2. **不要跳过西西喵斯直接调用子 Agent** - 会破坏工作流编排
3. **新增资料后更新数据库** - 通过 AI 操作数据库，Markdown 自动同步
4. **无构建脚本** - 本仓库没有 `package.json` 脚本、测试框架或 CI/CD
