# 中央调度智能体实现说明

## 概述
中央调度智能体（学习管家）是喵学堂系统的核心协调器，负责理解用户意图并分派任务给相应的领域专家智能体。

## 已实现组件

### 1. 技能系统
已创建以下Claude技能，可通过Skill工具调用：

| 技能名称 | 英文标识 | 功能描述 |
|----------|----------|----------|
| 学习管家 | learning-dispatcher | 中央调度器，识别意图并分派任务 |
| 图书管理员 | learning-librarian | 知识检索与解释 |
| 授课老师 | learning-teacher | 系统化讲解与答疑 |
| 出题考官 | learning-examiner | 练习生成与评估 |
| 学习伙伴 | learning-companion | 情感支持与动机维护 |
| 学情分析师 | learning-analyst | 学习数据分析与建议 |

### 2. 意图分类映射
中央调度器根据用户输入分类意图，并映射到相应技能：

```
用户输入 → 意图识别 → 技能选择 → 任务执行 → 结果整合
```

**意图类型与技能映射**：
- **知识查询** → `learning-librarian`
- **学习讲解** → `learning-teacher`
- **练习测试** → `learning-examiner`
- **进度回顾** → `learning-analyst`
- **情感支持** → `learning-companion`
- **系统配置** → (待实现)

### 3. 使用方式
中央调度智能体可通过以下方式工作：

#### 方式一：自动识别与分派（推荐）
用户直接与系统交互，中央调度器自动识别意图并调用相应技能：
```
用户: "我想学习微积分"
→ 中央调度器识别为"学习讲解"意图
→ 调用`learning-teacher`技能
→ 返回系统化的微积分讲解
```

#### 方式二：手动指定技能
用户可直接指定使用特定技能：
```
用户: "/learning-teacher 请讲解微积分"
→ 直接调用授课老师技能
→ 返回讲解内容
```

## 技术实现

### 技能触发机制
每个技能都有明确的触发条件描述，Claude会根据这些描述自动判断何时使用哪个技能。

### 上下文传递
技能间可共享上下文信息，确保连贯的学习体验：
1. 学习进度和水平信息
2. 已学习的知识点
3. 历史交互记录
4. 个人偏好设置

### 工作流编排
对于复杂学习任务，中央调度器可协调多个技能：
```
用户: "我想学习量子力学，先了解基本概念，然后做练习题"
→ 调用`learning-librarian`获取基础概念
→ 调用`learning-teacher`进行详细讲解
→ 调用`learning-examiner`生成练习题
→ 整合所有结果返回给用户
```
